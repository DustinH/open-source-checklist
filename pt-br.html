Open Source Checklist

Um guia para ajudar você a lembrar de coisas importantes na hora de criar seus projetos abertos no Github ;D

Eu quero contribuir!

Tarefas

Escolha um nome descolado!
Crie uma descrição coerente para seu projeto.
Adicione um README.md contendo todas as informações básicas sobre o seu projeto.
Crie um arquivo CONTRIBUTING.md contendo todas as informações necessárias para que outros desenvolvedores ajudem ;)
Adicione um arquivo .gitignore contendo tudo o que você não deseja versionar.
Escolha uma licença adequada.
Escolha uma metodologia de versionamento e a siga!
Defina um Roadmap para deixar claro os rumos que o projeto vai tomar.
Documente seu projeto da melhor forma possível.(Como usar, dependências, suporte, etc...)
Descreva suas Releases de forma clara e objetiva ou crie um arquivo CHANGELOG.md e faça isso lá.
Use Badges para indicar os vários status do projeto.
Configure e adicione um .editorconfig.
Quando necessário, crie um website para seu projeto. Aproveite e publique com github pages ;)
Escreva testes automatizados!
Integre seus testes a um serviço de Integração Contínua.
Disponibilize seu projeto no gerenciador de pacotes adequado. (EX: bower, npm, etc...)

Dicas e Truques

Aprenda tudo sobre o Github.
Seja o mestre em Markdown.
Se você é estudante, receba seu Student Developer Pack.
Use Emoji Emoticons sem moderação.
Use sempre SVG.
Lembre-se da existência do Gitter para integrar um Chat a seu repositório.
Submeta seu projeto a newsletter adequada.
Submeta seu projeto a awesome list adequada.
Artigo com mais dicas iradas para criar seus projetos abertos focando em javascript :)
